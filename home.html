A/B Testing

Introduction

How do designers decide between two or more versions of the same design? The answer lies in A/B Testing. In this project, I will use A/B Testing and Statistical Testing methods to gauge the difference between two iterations of the same website.
Click here to access the webpages.

The Webpages

Webpage A

Webpage B

For this project, I designed two versions of a shopping page for a cacti shop.

In Version A, the product cards, or divs, are larger, taking up two per row. there is also greater distance between them. The product information is displayed in a vertical orientation, with the price, description, ‘Add to Cart’ button, and cart count taking up one line each. Finally, the Cart button is aligned with the left edge of the product cards container.

In Version B, the product containers are smaller, and the ‘Add to Cart’ button is on the same line as the cart count. The price is also added to the end of the description. The Cart button is also more flushed against the left margin.


Tests

The Task:

Add over $150 of cacti to your cart and check out.

Test 1: Time to Completion


For the first test, I wanted to measure how long it takes users to complete the test on each website, and determine if there is a significant difference between the two. In this case, because I would be measuring the average time spent on the sites by each user, I used a T-Test.

Null Hypothesis

Between Webpage A and Webpage B, there is no significant difference in the time taken to complete the task.

Alternative Hypothesis

Between Webpage A and Webpage B, there is a significant difference in the time taken to complete the task.


Test 2: Return Rate

For the second test, I wanted to measure how many returns to the shop page the user makes from the Check Out page, and see if there is a significant difference between the return rates of the two sites. Because I would be recording the categories of users into did return and did not return, I used a chi-squared test.

Null Hypothesis

Between Webpage A and Webpage B, there is no significant difference in the number of returns to a page to complete the task.

Alternative Hypothesis

Between Webpage A and Webpage B, there is a significant difference in the number of returns to a page to complete the task.


Collecting the Data

To collect the data, I set up an app on Heroku that, upon launching, would randomly load either webpage A or webpage B. Then, my classmates went onto the app, either landing on page A or B, and completed the task. They would first add cacti to the cart, and then proceed to the check out page.

To analyze the data, I pulled from Heroku a log of all the activity that occured on the App. The log displayed the timestamp of each event, which version the event occured on, and the user that completed the event. By hand, I went through the log, pulling out the duration that each user spent on the site, as well as whether they did any returns to the initial page after going to the Check out page or not. This data was categorized either under page A, or page B.

Takeaways

The limitations I faced in AB Testing:
- Not many users to gain feedback from; only around 23 users
- Number of users I had between Webpage A and B was skewed: 15 for A, and only 8 for B

Takeaways on Design Principles:
- Size does not have as much as an impact as layout does in a website
- As long as a button is clearly made as a Call-to-Action, the user will search for it regardless of layout and hierarchy.
